Frequently Asked Questions

What is MediaPipe to Blender?

MediaPipe to Blender is an add-on for Blender that enables real-time animation of armature bones using MediaPipe's motion capture capabilities. It captures face, hand, and body landmarks from a webcam and maps them to Blender armature bones for live animation.

What are the system requirements?

The add-on requires:

  • Blender 2.93 or newer (Blender 4.0+ recommended)
  • Python 3.7 or newer (included with Blender)
  • A webcam or video input device
  • 4GB RAM minimum (8GB+ recommended)
  • Windows 10/11, macOS 10.15+, or Linux

How do I install the add-on?

  1. Download the add-on ZIP file from the download page.
  2. In Blender, go to Edit > Preferences > Add-ons.
  3. Click "Install..." and select the downloaded ZIP file.
  4. Enable the add-on by checking the box next to "Animation: MediaPipe Motion Capture".

The first time you use the add-on, it will automatically install required Python packages.

Do I need special equipment for motion capture?

No, you only need a standard webcam. MediaPipe to Blender uses computer vision and machine learning to detect landmarks from regular video input, eliminating the need for specialized motion capture equipment.

Can I use the add-on with my custom character rig?

Yes, the add-on supports custom character rigs. You can use:

  • Auto mapping, which works best with standard bone naming conventions
  • Preset mappings for popular rigs like Rigify and Mixamo
  • Manual mapping for complete control over how landmarks map to your specific bones

How accurate is the motion capture?

The accuracy depends on several factors:

  • Webcam quality and resolution
  • Lighting conditions
  • Distance from the camera
  • Complexity of the movements

MediaPipe provides good accuracy for most common movements and expressions. The add-on also includes smoothing options to reduce jitter and improve the quality of the animation.

Can I edit the recorded animations?

Yes, the add-on creates standard Blender animation data, which means you can:

  • Edit the recorded animation in the Dope Sheet or Graph Editor
  • Apply Blender's animation modifiers and constraints
  • Mix with other animation techniques
  • Export to any format Blender supports

Is the add-on free to use?

Yes, MediaPipe to Blender is completely free and open-source. It is released under the MIT License, which allows for both personal and commercial use without restrictions.

How can I improve performance?

To improve performance:

  • Reduce camera resolution in the add-on preferences
  • Disable features you don't need (e.g., face detection if only animating body)
  • Increase keyframe interval
  • Increase smoothing value
  • Close other applications to free up resources
  • Use a computer with a dedicated GPU

Can I contribute to the project?

Yes, contributions are welcome! The project is hosted on GitHub, and you can contribute by:

  • Reporting bugs
  • Suggesting new features
  • Submitting pull requests
  • Improving documentation
  • Creating tutorials or examples

Visit the GitHub repository to get started.

Troubleshooting Guide

Installation Issues

Problem: Add-on fails to install

Blender shows an error when trying to install the add-on.

Solution:

  1. Make sure you're using Blender 2.93 or newer
  2. Check if the ZIP file is corrupted (try downloading again)
  3. Try installing from the Blender preferences instead of drag-and-drop
  4. Check Blender's console for specific error messages

Problem: Python packages fail to install

The add-on shows errors about missing MediaPipe or other packages.

Solution:

  1. Ensure you have an internet connection
  2. Try manually installing the packages: 
    pip install mediapipe opencv-python pyzmq
  3. Check if your Python version is compatible (3.7+)
  4. Try running Blender with administrator privileges

Connection Issues

Problem: Cannot connect to camera

The add-on cannot access the webcam.

Solution:

  1. Ensure your webcam is connected and working
  2. Check if another application is using the webcam
  3. Try a different camera index in the add-on preferences
  4. Check webcam permissions in your operating system
  5. Restart Blender after connecting the webcam
